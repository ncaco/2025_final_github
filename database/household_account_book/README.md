# 가계부 기능 기획 및 설계

## 📋 프로젝트 개요

가계부 기능은 사용자의 일상적인 수입과 지출을 체계적으로 기록하고 관리할 수 있는 개인 금융 관리 시스템입니다. 직관적인 UI/UX와 강력한 분석 기능을 통해 사용자들이 재정 상태를 효과적으로 파악하고 관리할 수 있도록 설계되었습니다.

## 🎯 주요 기능

### 기본 기능
- ✅ 거래 등록/수정/삭제 (수입/지출)
- ✅ 카테고리별 분류 및 계층 구조
- ✅ 기간별 거래 조회 및 필터링
- ✅ 수입/지출 합계 및 잔액 계산

### 고급 기능
- ✅ 통계 및 차트 분석
- ✅ 데이터 내보내기 (CSV/Excel)
- ✅ 반응형 웹 디자인
- ✅ 실시간 데이터 동기화

### 향후 확장 기능
- 🔄 예산 관리 및 알림
- 🔄 반복 거래 자동 등록
- 🔄 영수증 이미지 인식 (OCR)
- 🔄 외부 금융 서비스 연동

## 🏗️ 시스템 아키텍처

### 기술 스택
- **Backend**: Spring Boot (Java) / PostgreSQL
- **Frontend**: Next.js (React/TypeScript) / Tailwind CSS
- **인증**: JWT 기반 인증
- **API**: RESTful API 설계

### 데이터베이스 설계
- **User**: 사용자 정보 관리
- **Category**: 계층적 카테고리 구조
- **Transaction**: 거래 내역 및 상세 정보
- **Account**: 계좌 정보 (선택사항)

## 📊 핵심 엔티티 관계

```
User (1) ──── (N) Category
  │
  └─── (N) Transaction (N) ──── (1) Category
```

## 🚀 개발 단계

### Phase 1: MVP 개발 (4주)
1. **Week 1**: 데이터베이스 설계 및 기본 API 구현
2. **Week 2**: 사용자 인증 및 기본 CRUD 기능
3. **Week 3**: 거래 관리 및 카테고리 시스템
4. **Week 4**: 기본 UI 구현 및 통합 테스트

### Phase 2: 고급 기능 (3주)
1. **Week 5**: 통계 및 차트 기능 구현
2. **Week 6**: 데이터 내보내기 및 필터링 고도화
3. **Week 7**: 성능 최적화 및 사용자 피드백 반영

### Phase 3: 확장 및 안정화 (2주)
1. **Week 8**: 추가 기능 구현 및 QA
2. **Week 9**: 배포 및 모니터링 설정

## 🔧 개발 환경 설정

### 필수 요구사항
- **Node.js**: 18.0 이상
- **Java**: 17 이상 (Backend 선택 시)
- **PostgreSQL**: 15 이상
- **Docker**: 컨테이너화 환경

### 로컬 개발 환경
```bash
# Frontend
cd frontend
npm install
npm run dev

# Backend (Spring Boot 선택 시)
cd backend
./gradlew bootRun

# Database
docker run -d -p 5432:5432 -e POSTGRES_PASSWORD=password postgres:15
```

## 📚 API 문서

상세한 API 엔드포인트 정보는 [`api_design.md`](./api_design.md)를 참조하세요.

### 주요 엔드포인트
- `POST /auth/login` - 사용자 로그인

- `POST /transactions` - 거래 등록
- `GET /statistics/summary` - 통계 요약 조회
- `GET /categories` - 카테고리 목록 조회
